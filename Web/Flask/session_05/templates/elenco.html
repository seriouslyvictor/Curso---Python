Como o elenco sobrevive entre requisicoes?

Cada clique no botao "Adicionar ao Elenco" e uma requisicao completamente nova, independente das anteriores. Entao como os jogadores anteriores ainda aparecem aqui?

Onde mora a lista
No nivel do modulo em app.py, fora de qualquer funcao. Python cria essa lista uma unica vez quando o servidor inicia.
Por que ela persiste
O servidor Flask e um processo Python que fica rodando continuamente. Ele nao "reinicia" a cada request — apenas executa a funcao da rota. A lista existe na memoria do processo e acumula os jogadores.
O que apaga a lista
Reiniciar o servidor (Ctrl+C e flask run de novo). A lista volta a ser vazia porque o processo Python recomeca.
Solucao permanente
Banco de dados. Na Sessao 6, os dados vao para o disco com SQLite e Flask-SQLAlchemy. Ai eles sobrevivem a qualquer reinicializacao.

Experimento: adicione alguns jogadores, depois pare o servidor com Ctrl+C no terminal e rode flask run de novo. O elenco vai sumir. Esse e o limite da memoria em Python — e a motivacao para a proxima sessao.
